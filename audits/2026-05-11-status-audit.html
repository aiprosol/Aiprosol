4 · Critical gaps · fix today

4.1 · /inbox is unreachable from anywhere on the site

Grepped src/ for href references to /inbox — the only matches are inside /inbox/page.tsx itself (self-links to its own Connect/Disconnect buttons). The page is not linked from the main nav, the footer, the BentoGrid, or any product page. Visitors can only reach it by typing the URL.

FixAdd a nav entry under "Products" → "Beta · Arora Inbox" pointing at /inbox, plus a card in src/components/BentoGrid.tsx. 5 min change.

4.2 · /inbox missing from sitemap.xml

src/app/sitemap.ts uses a hardcoded staticRoutes array. /inbox is not in it. Google won't crawl the beta page, and the sitemap is otherwise comprehensive (30 entries).

FixAppend '/inbox' to the staticRoutes array in src/app/sitemap.ts with priority: 0.5 (low — beta).

4.3 · Replies to hello@aiprosol.com will bounce

Verified via dig +short MX aiprosol.com — no MX records on the apex domain. The ROI report email sends via Resend (verified), but if a customer hits "Reply" they bounce. SPF and DKIM TXT records are also empty on the apex (Resend likely set them on a sending subdomain like send.aiprosol.com).

FixAdd free ImprovMX forwarding: hello@aiprosol.com → srijanpaudel219@gmail.com. Two MX records, ~3 minutes in the Vercel DNS panel.

5.2 · SESSION_SECRET unset in production

Vercel env list shows no SESSION_SECRET. src/lib/auth.ts falls back to a hardcoded dev secret and logs '[auth] SESSION_SECRET unset in production — using fallback. ROTATE IMMEDIATELY.' This affects: magic-link sign-in, the OAuth state JWT, and the Arora Inbox Gmail cookie. Anyone with the public source code can forge cookies.

Fixprintf "$(openssl rand -hex 32)" | npx vercel env add SESSION_SECRET production · then redeploy. 30 sec.

5.3 · Stripe keys unset · /api/checkout returns 503

STRIPE_SECRET_KEY, STRIPE_PUBLISHABLE_KEY, STRIPE_WEBHOOK_SECRET all empty locally and on Vercel. Plan subscriptions on /pricing route through /api/checkout which returns 503 and redirects to /contact. This was an explicit early decision — Gumroad handles all products under $1K; Stripe is only needed for the $497/$1497 monthly plans. Noting it here so it's not forgotten.

5.4 · GROQ_API_KEY unset · Arora chat falls back to canned replies

Confirmed graceful degradation in src/app/api/arora-chat/route.ts:54–61 — when key is missing, returns fallbackReply() with mode: 'fallback'. Customers still get answers, just templated ones. Detection: detectIntent() handles the routing.
