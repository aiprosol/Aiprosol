Each product folder contains:
- `README.md` — title, audience, what's inside, Gumroad URL, deliverable path
- `delivery/` — actual file(s) to upload to Gumroad (HTML/PPTX/XLSX)
- `source/` — markdown sources (where applicable)

## Daily upload plan (~30 min)

Open Finder once: `Cmd+Shift+G` → paste `/Users/user/Airprosol/products catalogue/` → Enter. Keep window visible.

Then for each of the 15 ready products:
1. Open the product's Gumroad admin URL (in each `README.md`)
2. Click **Content** tab
3. Drag the file from `<product>/delivery/` into the upload area
4. Click **Save changes** → toggle **Publish and continue** (green)

For the 3 bundles, upload all listed files into a single product's Content tab.

After all are published, fire a $1 test purchase end-to-end to prove the flow.

## Where things live (architecture map)

| Layer | What | Where |
|---|---|---|
| Source content | Markdown sources | `<product>/source/*.md` |
| Delivery files | Files to upload to Gumroad | `<product>/delivery/*` |
| Per-product description | Audience + what's inside + admin URL | `<product>/README.md` |
| Web app product cards | Title + price + description + buyUrl | `web/src/content/products.json` |
| Buy button routes | aiprosol.com `→` Gumroad | wired via `buyUrl` in `products.json` |
| Payment processing | Gumroad — handles cards + tax + delivery email | gumroad.com |
| Analytics on purchases | PostHog `product_checkout_clicked` events + Gumroad sales tab | eu.posthog.com + gumroad.com |
